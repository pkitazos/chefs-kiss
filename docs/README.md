## Contributing to Docs

When adding new features or making significant changes:

1. Create or update the relevant documentation file in `/docs`
2. Add a link to it in this README
3. Update the main [README.md](../README.md) if needed
4. Keep examples up-to-date with actual code

## Documentation Standards

- Use clear, concise language
- Include code examples for all features
- Explain the "why" not just the "how"
- Add troubleshooting sections where relevant
- Keep file names lowercase with hyphens (kebab-case)
